# Как давать промпты для улучшения проекта

## Золотое правило

**Чем конкретнее запрос — тем точнее результат.**

---

## 1. Структура хорошего промпта

### Формула:
```
[ЧТО сделать] + [ГДЕ именно] + [КАК должно работать] + [ОГРАНИЧЕНИЯ]
```

### Пример плохого промпта:
> "Сделай чтобы было красивее"

### Пример хорошего промпта:
> "На странице player.html — карточки инвентаря слишком мелкие на мобильном.
> Сделай их шире (минимум 200px), увеличь шрифт названия до 0.9rem,
> и добавь иконку рарности слева от имени предмета."

---

## 2. Типы запросов и как их формулировать

### 🐛 Баг-репорт
```
Что происходит: [описание бага]
Что должно происходить: [ожидаемое поведение]
Как воспроизвести: [шаги]
Страница/файл: [где именно]
Ошибка в консоли (если есть): [текст ошибки]
```

**Пример:**
> Когда игрок покупает предмет в магазине, золото списывается но предмет
> не появляется в инвентаре. Консоль показывает "TypeError: Cannot read
> property 'item_id' of undefined". Это на странице player.html в модалке магазина.

### ✨ Новая фича
```
Что добавить: [описание функции]
Где это должно быть: [страница / панель / раздел]
Кто использует: [GM / игрок / оба]
Как взаимодействует с существующим: [связи с другими фичами]
Пример использования: [сценарий]
```

**Пример:**
> Добавь систему заметок для GM. В GM-панели новая вкладка "Notes".
> GM может создавать заметки с заголовком и текстом, привязывать их
> к персонажам. Заметки сохраняются в БД и доступны между сессиями.
> При наведении на персонажа в списке — показывать последнюю заметку.

### 🎨 Визуальные изменения
```
Что изменить: [элемент]
Где: [страница / селектор / скриншот]
Как сейчас: [текущее состояние]
Как должно быть: [желаемое состояние]
Референс (если есть): [ссылка на пример]
```

**Пример:**
> HP-бар на странице игрока — сделай его выше (с 8px до 16px),
> добавь числа прямо внутри бара (белый текст с тенью),
> и плавную анимацию при изменении HP (0.5s transition).

### ⚡ Оптимизация / рефакторинг
```
Что тормозит / что плохо: [описание проблемы]
Где заметил: [конкретное место]
Насколько критично: [блокирует работу / просто раздражает]
```

### 🔧 Изменение существующего
```
Что сейчас: [текущее поведение]
Что хочу вместо этого: [новое поведение]
Почему: [причина изменения]
```

## 3. Что мне ОЧЕНЬ помогает

### ✅ Скриншоты
Если что-то выглядит не так — скриншот лучше 100 слов.
Просто вставь скриншот в чат.

### ✅ Ошибки из консоли
Если что-то не работает — открой DevTools (F12) → Console, скопируй ошибку.

### ✅ Конкретные файлы
Если знаешь где проблема — укажи файл:
> "В файле gm-app.js функция refreshChars — она не обновляет HP после атаки"

### ✅ Приоритеты
Если запрос большой — скажи что важнее:
> "Сначала сделай X, потом Y. Z — можно потом."

### ✅ Примеры / референсы
> "Хочу как в Roll20 — когда наводишь на токен, показывается попап с инфой"

---

## 4. Чего лучше избегать

| ❌ Не делай так | ✅ Лучше так |
|---|---|
| "Сделай лучше" | "Увеличь контраст текста, добавь hover-эффекты на кнопки" |
| "Почини всё" | "При нажатии кнопки X ничего не происходит на странице Y" |
| "Добавь фичу как в D&D Beyond" | "Добавь список заклинаний: имя, уровень, описание, кнопка 'Использовать'" |
| "Сделай норм дизайн" | "Используй стиль dark fantasy из base.css, золотые акценты, закруглённые карточки" |
| Несколько разных задач в одном сообщении | Одна задача = одно сообщение (или чётко пронумеруй) |

## 5. Шаблоны для частых запросов

### Добавить новую страницу:
```
Создай страницу [название] по адресу /[url].
Она должна содержать: [список секций].
Стиль: как остальные страницы (dark fantasy, base.css).
Данные берёт из API: [какие эндпоинты].
Доступна для: [GM / игрок / все].
```

### Изменить модель данных:
```
Добавь поле [имя поля] (тип: [тип]) в модель [Character/Session/Item].
Значение по умолчанию: [X].
Показывать: [где в UI].
Редактировать: [кто может, где].
```

### Добавить WebSocket событие:
```
Новый WS event: [имя события].
Когда триггерится: [условие].
Данные в событии: [поля].
Кто получает: [все / GM / конкретный игрок].
Что делает на фронте: [описание реакции UI].
```

## 6. Мультизадачный промпт (если задач много)

Нумеруй и расставляй приоритеты:

```
Нужно сделать 3 вещи:

1. [КРИТИЧНО] Баг: при входе в сессию иногда показывает пустой экран.
   Консоль: "Cannot read session_code of null". Страница player.html.

2. [ВАЖНО] Добавить кнопку "Бросить кубик" на страницу игрока.
   d20 бросок, результат показывать в логе и отправлять через WS всем.

3. [ПОТОМ] Поменять шрифт заголовков на что-то более фэнтезийное.
```

---

## 7. Итерация и обратная связь

После каждого изменения:
- **Работает?** → "Ок, дальше" или "Продолжай"
- **Не работает?** → Скриншот + ошибка из консоли
- **Работает, но не то** → "Работает, но хочу чтобы [конкретное отличие]"
- **Хочу откатить** → "Верни как было в [файл], последнее изменение не нужно"

## Структура проекта (для справки)

```
dnd-companion/
├── main.py                  # Точка входа, роуты страниц, настройки
├── config.json              # API ключи, настройки сервера
├── ai_system_prompt.txt     # Промпт для AI ассистента ARIA
├── app/
│   ├── models.py            # Все модели БД (Session, Character, Item...)
│   ├── schemas.py           # Pydantic схемы валидации
│   ├── database.py          # Подключение к SQLite
│   ├── game_mechanics.py    # Формулы урона, атаки, лечения
│   ├── ai_agent.py          # AI контекст и OpenRouter интеграция
│   ├── websocket_manager.py # Менеджер WS подключений
│   └── routers/
│       ├── sessions.py      # Создание/подключение к сессиям
│       ├── characters.py    # CRUD персонажей, эффекты, модификаторы
│       ├── combat.py        # Расчёты боя (урон, атака, лечение)
│       ├── initiative.py    # Порядок хода
│       ├── map.py           # Карта, токены, туман войны
│       ├── inventory.py     # Предметы, инвентарь, магазин
│       ├── ai.py            # AI чат эндпоинты
│       └── websocket.py     # WS подключение и события
├── static/
│   ├── lobby.html           # Главная — создать/присоединиться
│   ├── gm.html              # Интерфейс GM
│   ├── player.html          # Лист персонажа игрока
│   ├── settings.html        # Страница настроек
│   ├── css/
│   │   ├── base.css         # Общие стили (dark fantasy тема)
│   │   ├── gm.css           # Стили GM страницы
│   │   └── player.css       # Стили страницы игрока
│   └── js/
│       ├── gm-app.js        # Логика GM интерфейса
│       ├── player-app.js    # Логика страницы игрока
│       ├── map-canvas.js    # Canvas карты (рендер, зум, токены, туман)
│       └── websocket-client.js  # WS клиент с реконнектом
└── data/
    ├── combat_companion.db  # База данных SQLite
    └── maps/                # Загруженные карты
```
